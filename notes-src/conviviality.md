after ivan illich. a tool is convivial if it enhances capability without creating dependency or counterproductivity. the test: does this tool make the person using it more capable, or does it make them dependent on it?

[[soft-stack-ecology]] is built on this filter. rssprint is convivial -- it runs locally, produces a file you own, requires nothing external. a saas newsletter platform is not: it holds your content, your audience, and your relationship to both.

conviviality is not about simplicity per se. a complex tool can be convivial if it enhances autonomy. a simple tool can be anti-convivial if it creates lock-in.
